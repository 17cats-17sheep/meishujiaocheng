# 命名规范

- 合理规划自己的资产编号。避免编号重复。
- 编号重复的资源会默认覆盖。

## 资源编号

- 请使用：资源类型_前缀+资源编号_后缀的命名方式，可有效避免相同命名资源覆盖造成的不必要麻烦。

- 前缀命名规则：

  | 资源类型 | 命名前缀 |
  | :------: | :------: |
  | 骨骼模型 |   SK_    |
  |   贴图   |    T_    |
  |   材质   |   MI_    |
  | 动画序列 |    A_    |

- 后缀命名规则：**开发者ID后5位**+**5位任意字母数字组合**

  - 例如：**00001+123ab，00001+h47q1，01593+xnez7，04892+6b2ry**

## 命名格式参考：

- 通用资源命名格式：
  - 模型命名：SK_ 风格_ 性别_ 部件名_ 资源编号
  - 材质命名：MI_ 风格_ 性别_ 部件名_ 资源编号
  - 贴图命名：T_ 风格_ 性别_ 部件名_ 资源编号
  - 物理命名：模型命名_ PhysicsAsset（模型命名正确，导入引擎后会自动生成）
  - 动画序列：A_ 风格_ 性别_ 资源编号
